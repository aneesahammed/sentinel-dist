Local-First Docs

Sentinel Documentation

Minimal docs for installing Sentinel, running it against Docker or local processes, and understanding what is documented now versus what still needs a fuller reference.

Sentinel is a local-first error tracker for development logs. It reads the logs you already have, groups recurring failures into issues, stores them locally in SQLite, and gives you a local UI to inspect what keeps coming back.

Documentation scope note This site is the start of the public docs, not the end state. Sentinel still needs a more extensive reference covering every command, subcommand, flag, UI state, filter, and troubleshooting path.

What is covered here now

  • installation and upgrades
  • quick start paths for Docker, wrapped commands, and log files
  • Docker-specific behavior, including sentinel docker list and sentinel docker explain
  • a compact command overview
  • the current local UI surface
  • OpenTelemetry setup notes

What still needs fuller documentation

  • every CLI command and subcommand with complete examples
  • all UI options, pages, filters, issue states, and actions
  • troubleshooting by runtime and log source
  • deeper OTEL and tracing guidance
  • a stronger release and versioning reference for maintainers
