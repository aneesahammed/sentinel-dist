Getting Started

Install

Get Sentinel running in one command and understand how upgrades work after the initial install.

One-command install

curl -fsSL https://github.com/aneesahammed/sentinel-dist/releases/latest/download/sentinel.sh | bash

The installer:

  • detects macOS or Linux and the current CPU architecture
  • downloads the matching release asset from aneesahammed/sentinel-dist
  • verifies the binary with checksums.txt
  • installs sentinel into a writable bin directory

Optional overrides

Install to a specific directory:

curl -fsSL https://github.com/aneesahammed/sentinel-dist/releases/latest/download/sentinel.sh | \
  env SENTINEL_INSTALL_DIR="$HOME/bin" bash

Pin to a specific published version:

curl -fsSL https://github.com/aneesahammed/sentinel-dist/releases/latest/download/sentinel.sh | \
  env SENTINEL_VERSION="0.1.0+42" bash

Verify the install

sentinel --help
sentinel check --offline

Upgrade later

After first install, upgrades should go through Sentinel itself:

sentinel update

Source build

If you are working from the repository instead of a published release:

make build

That installs a local binary with version metadata baked in.
