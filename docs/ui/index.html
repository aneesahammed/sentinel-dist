Local UI

UI

What the local Sentinel UI does today and which UI areas still need fuller documentation.

Start the UI

sentinel ui

The UI runs locally and reads from the same SQLite-backed Sentinel data on your machine.

Current UI surface

Today the UI is centered around issue inspection and triage:

  • issue list and project filtering
  • issue detail pages
  • occurrence selection
  • tags, frames, breadcrumbs, and event context
  • timeline views and basic activity spikes
  • issue state actions like resolve, investigate, suppress, and reopen

What still needs fuller UI docs

The docs still need a more complete UI reference covering:

  • all screens and navigation paths
  • filter behavior
  • issue states and transitions
  • keyboard shortcuts
  • settings and local defaults
  • troubleshooting common local UI issues

Supporting docs already in the repo

Until the docs site grows further, these repository docs still matter:

  • docs/keyboard-shortcuts.md
  • README.md

Those should be folded into this docs site over time instead of remaining scattered.
